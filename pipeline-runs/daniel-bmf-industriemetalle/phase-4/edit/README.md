## Premiere Import — Step by Step

### Option A — FCP7 XML (recommended, imports all 118 markers at once)

1. Open Premiere Pro 2024+
2. Create or open the project `daniel-bmf-industriemetalle.prproj`
3. Import Daniel's master talking-head clip into the project bin (name it something like `Daniel_Master.mov`)
4. Drag it onto the timeline to create a sequence
5. Rename the sequence to match: `daniel-bmf-industriemetalle-markers`
6. Set the sequence to **30fps, 1920×1080, square pixels**
7. `File → Import` → select `premiere-markers.xml`
8. In the import dialog choose: **"Create new sequence with markers"** or **"Import markers into existing sequence"**
9. All 118 markers now appear in the Marker panel (`Window → Markers`)

**Marker categories (filter via Marker panel):**
- `CUT-XXX` — 71 cut points (also annotated as J12f/L15f for overlaps)
- `ZOOM-XX` — 18 zoom instructions (range markers show duration)
- `PI-XX` — 18 pattern interrupts
- `BROLL-XX` — 11 B-Roll slot ranges

### Option B — CSV Marker Import (cuts only)

If only the 71 cuts are needed:
1. `Window → Markers` to open the Markers panel
2. Use plugin **"CSV Marker Import"** (free, by Adobe Community) OR manually
3. Some Premiere workflows: `File → Import` CSV is NOT native — use `Motion Array Marker Importer` or paste via AppleScript

CSV columns follow the Adobe Premiere marker export format:
`Marker Name, Description, In, Out, Duration, Marker Type`

### Option C — Manual placement from JSON
If XML/CSV import isn't working, open `cut-points.json` and use the `tc` field (SMPTE timecode) to place markers manually. The frame field works too if your timeline is set to frame-counting display.

## Zoom Keyframe Application

Zoom values are for **Premiere's Motion → Scale effect** (percent values) or **Lumetri Transform**:

| JSON field | Premiere equivalent |
|---|---|
| `scale` (e.g., 1.12) | Motion → Scale = **112** |
| `transform_origin: "50% 45%"` | Motion → Anchor Point Y = 486 (540 × 0.9) |
| `transform_origin: "50% 40%"` | Motion → Anchor Point Y = 432 |
| `transform_origin: "50% 38%"` | Motion → Anchor Point Y = 410.4 |
| `easing: Easing.inOut(Easing.cubic)` | Right-click keyframe → "Ease In & Out" → Bezier 0.4/0/0.2/1 |
| `easing: Easing.out(Easing.expo)` | Right-click keyframe → "Ease Out" → Bezier 0.19/1/0.22/1 |

**Note:** The transform_origin math assumes 1920×1080. For a 50% X origin the Anchor Point X stays at 960.

## Cut Type Implementation Notes

### J-Cuts (6 cuts)
Audio of the incoming clip leads the video by `overlap_f` frames.
- In Premiere: Unlink audio from video on the incoming clip, drag audio left by overlap frames.
- Apply crossfade **only on audio** (4-6f constant-power) to avoid pop.

### L-Cuts (10 cuts)
Audio of the outgoing clip trails the video by `overlap_f` frames.
- In Premiere: Unlink audio from video on the outgoing clip, extend audio right by overlap frames into next clip.
- Apply 4-6f audio fade on the trailing tail.

### Hard Cuts (55 cuts)
Standard frame-exact cut. No audio/video unlink needed.

## Pattern Interrupt → Overlay Integration

The 18 pattern interrupts are integrated with **Phase 3B overlays** (from master-timeline `phase_3_assignments.3B_mograph.approved_overlays`). Most PIs have `linked_overlay_id` fields pointing to the ovl-XXX that carries the motion graphics payload.

**Handled by Remotion (not Premiere):**
- Fullscreen takeovers (ovl-005, ovl-015, ovl-024) — build in Remotion, export as PNG sequence or PreMultiplied MOV, import into Premiere track V3
- KineticText, StatCards, Timeline etc. — same (Remotion render → Premiere overlay)
- Letterbox — can be either Remotion or Premiere native (use Premiere Crop effect 10% top+bottom)

**Handled by Premiere directly:**
- Hard cuts, J-cuts, L-cuts, zooms (Motion panel)
- Letterbox with Crop Effect
- Color shifts (Lumetri Color)
- Freeze frames (right-click → Add Frame Hold)
